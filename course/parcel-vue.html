            【parcel】打包vue项目

            太阳丶 2018-8-23 parcel

            webpack 打包的高性能给我们的项目构建带来了便捷，同时复杂的配置也让我们苦不堪言，这个时候parcel横空出世，号称“零配置“。在这里做一个笔记，记录如何用parcel打包一个vue.js项目。

            # 什么样的情况下使用parcel

            parcel基本不需要配置，用于构建中小型项目时的速度非常快，推荐代码行在15k以下可以使用parcel做构建工具。

            # 新建一个vue项目

            项目结构可以使用vue-cli生成的src目录项目结构。 初始化项目:

            npm init

            修改package.json文件中的入口文件：

            "main": "main.js",

            添加快捷命令：

            "scripts": {
  "dev": "parcel index.html -p 8089 --open",
	"build": "parcel build index.html --public-url ./"
},
            注意
            • 这里的-p 8089 是设置运行服务器端口，不写则端口默认为1234。
            • --open 是运行完成后自动打开浏览器，不写则不会自动打开。
            • 打包不能缺少--public-url ./，否则打包后的静态文件路径会找不到。

            # 安装依赖

            安装vue和parcel需要的依赖模块

            // 安装parcel，这里把parcel安装到项目中，全局的parcel打包时会出现错误。
npm install parcel-bundler --save-dev

// 安装vue和vue-router
npm install vue vue-router --save

// 安装babel转码
npm install babel-preset-env --save

// 安装.vue文件解析插件,如果parcel的版本大于1.7.0，也可以不安装
npm install parcel-plugin-vue --save

            新建.babelrc文件，添加配置：

            {
    "presets": [
        "env"
    ]
}

            如果要使用PostCss。新建postcss.config.js文件，添加配置：

            module.exports = {
  plugins: [
    require('autoprefixer')({ browsers:  [
                              'last 20 versions',
                              'IE 9',
                              'iOS >= 8']})]
}

            使用less或者Sass，则安装：

            npm install less --save

npm install node-sass --save

            其他的 vuex，axios，element-ui等插件按常规方法一一安装即可。

            # Command

            • npm run dev 运行
            • npm run build 构建 有可能出现的错误:
            Server running at http://localhost:1234
× C:\Users\YunYou\Documents\GitHub\vue-parcel-demo\src\router\index.js:3:23: Cannot resolve dependency ‘@/components/HelloWorld’

            原因是webpack可以设置路径别名@，把项目文件中导入路径修改成相对路径即可。

            [Vue warn]: You are using the runtime-only build of Vue where the template compiler is not available. Either pre-compile the templates into render functions, or use the compiler-included build.(found in )

            在使用vue-cli生成webpack模板过程中，有如下提示：

            ? Vue build (Use arrow keys)
> Runtime + Compiler: recommended for most users
  Runtime-only: about 6KB lighter min+gzip, but templates (or any Vue-specific HTML) are ONLY allowed in .vue files - render functions are required elsewhere

            如果选择 Runtime + Compiler, main.js 为

            new Vue({
  el: '#app',
  router,
  components: { App },
  template: '<App/>'
})

            如果选择 Runtime-only, main.js 则为

            new Vue({
  el: '#app',
  router,
  render: h => h(App)
})

            Parcel 使用 runtime-only, 修改 main.js 中内容为 Runtime-only 形式 render: h => h(App) 即可。

            # parcel-vue

            parcel-vue是 一个基于parcel打包工具的 vue开发脚手架解决方案,强烈建议使用node8.0以上。项目地址

            # 初始化项目

            // 安装脚手架
$ npm install parcel-vue -g
// 初始化
$ parcel-vue projectName

            # 安装依赖

            $ cd projectName
$ npm install

            # 运行构建

            npm run dev

npm run build

            # 路由懒加载

            只需将传统 import page from ‘path’ 方式改为 const page = () => import(‘path’) 即可

            // 此种方式路由不会懒加载
import HelloWorld from '../views/HelloWorld/HelloWorld.vue'
// 此种方式引入即可实现路由懒加载,打包时js文件自动拆分
const HelloWorld =  () => import('../views/HelloWorld/HelloWorld.vue')
