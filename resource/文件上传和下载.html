文件上传和下载方式整理

太阳丶 2019-9-23 html5

# 文件上传

# 1. 通过 form 表单上传

<form action="uploadurl" enctype="multipart/form-data" type="post">
  <input type="file" />
  <input type="hidden" name="userid" />
  <input type="hidden" name="signature" />
  <button>提交</button>
</form>
  • 优点：使用简单方便，兼容性好，基本所有浏览器都支持
  • 缺点：无法监控上传状态，提交数据后页面会跳转，当长传参数较多时，会造成 dom 冗余

# 2. 通过 formData 对象上传

使用 js 构造表单数据，只兼容 IE9 以上

export const upload = params => {
  const formData = new FormData()
  Object.keys(params).forEach(key => formData.append(key, params[key]))
  axios.post("url", formData, {
    headers: {
      "Content-Type": "multipart/form-data" // 注意要设置Content-Type，原生ajax也是如此
    }
  })
}

# 3. 使用 fileReader 读取文件数据进行上传

HTML5 的新 api，兼容性也不是特别好，只兼容到了 IE10

var fr = new FileReader()
fr.readAsDataURL(file)
fr.onload = function(event) {
  var data = event.target.result //此处获得的data是base64格式的数据
  img.src = data
  ajax(url, { data }, function() {})
}
  • 优点：同 formData 上传，可以预览上传文件
  • 缺点：一次性发送大量的 base64 数据会导致浏览器卡顿，服务器端接收这样的数据可能也会出现问题

# 4. 分片上传

使用 FileReader 读取文件，将文件切割成多个部分，分次上传

export const spliceUpload = (file, cb, size = 1024 * 1024) => {
  let reader = new FileReader()
  reader.readAsArrayBuffer(file)
  reader.addEventListener("load", () => {
    const data = []
    const [name, ext] = file.name.split(".")
    for (let i = 0, len = this.result.byteLength; i < len; i += size) {
      data.push(this.result.slice(i, i + size))
    }
    cb({ name, ext, data }) // 接收分片数据回调函数
  })
}

# 文件下载

# 1. 通过 window.open()打开新页面下载文件

window.open("文件url", "_self")
  • 优点：简单粗暴
  • 缺点：当参数错误时，无法监听接口返回的错误信息，下载视频和音频时浏览器会直接播放，无法下载

# 2. 通过 a 标签打开新页面下载

// 通过js动态创建a标签下载
export const downloadFile = (url, fileName) => {
  const link = document.createElement("a")
  const body = document.querySelector("body")

  link.href = url
  link.download = fileName

  // fix Firefox
  link.style.display = "none"
  body.appendChild(link)

  link.click()
  body.removeChild(link)
}

<!-- 静态标签直接下载 -->
<a href="文件url" download="重命名">点击下载</a>
  • 优点：简单粗暴，可以自定义文件名称
  • 缺点：当参数错误时，无法监听接口返回的错误信息，下载视频和音频时浏览器会直接播放，无法下载

# 3. 通过表单提交下载

export const downloadFile = option => {
  const { type = "post", url, param } = option
  const form = document.createElement("form")
  form.action = url
  form.method = type
  // form.target = '_blank';
  Object.keys(param).forEach(item => {
    const input = document.createElement("input")
    input.type = "hidden"
    input.name = item
    input.value = param[item]
    form.appendChild(input)
  })
  document.body.appendChild(form)
  form.submit()
  document.body.removeChild(form)
}
  • 优点：浏览器兼容性好
  • 缺点：过程繁琐，无法监听接口返回的错误信息

# 4. 通过 Blob 或 ArrayBuffer 下载

Blob

Blob 对象表示一个不可变、原始数据的类文件对象。Blob 表示的不一定是 JavaScript 原生格式的数据。File 接口基于 Blob，继承了 blob 的功能并将其扩展使其支持用户系统上的文件。

ArrayBuffer

ArrayBuffer 对象用来表示通用的、固定长度的原始二进制数据缓冲区。ArrayBuffer 不能直接操作，而是要通过类型数组对象或 DataView 对象来操作，它们会将缓冲区中的数据表示为特定的格式，并通过这些格式来读写缓冲区的内容。

理解 DOMString、Document、FormData、Blob、File、ArrayBuffer 数据类型

// 通过原生ajax请求返回Blob对象
export const downloadFile = () => {
  return new Promise(resolve => {
    const xhr = new XMLHttpRequest()
    xhr.open("method", "url", "async")
    xhr.setRequestHeader("Content-type", "application/x-www-form-urlencoded")
    xhr.responseType = "blob"
    xhr.onreadystatechange = () => {
      if (xhr.readyState === 4 && xhr.status === 200) {
        resolve(xhr.response)
      }
    }
    xhr.send("params" || null)
  })
}
// 读取Blob生成下载地址
export const saveFile = blob => {
  if (window.navigator.msSaveOrOpenBlob) {
    // 兼容ie
    navigator.msSaveBlob(blob)
  } else {
    const link = document.createElement("a")
    const body = document.querySelector("body")

    link.href = window.URL.createObjectURL(blob) // 创建对象url
    // fix Firefox
    link.style.display = "none"
    body.appendChild(link)
    link.click()
    // link.dispatchEvent(new MouseEvent('click))
    body.removeChild(link)
    window.URL.revokeObjectURL(link.href) // 释放 URL 对象
  }
}

# 5. 实现批量打包下载

批量下载可以多次调用 4downloadFile实现，但是文件是单个的，这样无法批量集中下载的文件。

实现批量，且打包下载多个文件需要用到两个库：jszip 和 file-saver,通过ajax获取文件，然后用jszip压缩文件，最后用 file-saver下载文件。

jszip: https://github.com/Stuk/jszip
file-saver: https://github.com/eligrey/FileSaver.js

export const getBlob = () => {} // 通过ajax获取文件
export const batchDownload = (urlArr, fileName = "打包文件") => {
  if (!urlArr.length > 0) return
  const zip = new JSZip()
  const cache = {}
  const promises = []
  urlArr.forEach(item => {
    const promise = getBlob(item.url).then(data => {
      // 下载文件, 并存成ArrayBuffer对象
      zip.file(item.fileName, data, { binary: true }) // 逐个添加文件
      cache[item.fileName] = data
    })
    promises.push(promise)
  })
  Promise.all(promises).then(() => {
    zip.generateAsync({ type: "blob" }).then(content => {
      // 生成二进制流
      FileSaver.saveAs(content, `${filename}.zip`) // 利用file-saver保存文件
    })
  })
}

注意

由于是浏览器做打包处理，文件过大或过多时，有可能导致打包速度慢或浏览器崩溃，实际生产中建议使用服务器打包
